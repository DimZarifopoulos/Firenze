# MyCityGuide - Android City Tour Application

MyCityGuide is an Android application designed to serve as a digital tour guide for city visitors. [cite_start]It highlights specific points of interest (POIs), provides interactive details, and engages users with educational content[cite: 342].

## Key Features

### 📍 Points of Interest (POIs)
* The app features at least 5 POIs, each including:
    * [cite_start]Title and Category (e.g., museum, monument, square)[cite: 348, 350, 353].
    * [cite_start]Detailed description and a representative image[cite: 349, 351, 352].
    * [cite_start]Geographical coordinates and a "Learn more" URL[cite: 354, 355].

### 🗺️ Map Integration
* [cite_start]An interactive map displays markers for all POIs[cite: 356, 357, 360].
* [cite_start]Clicking on a marker navigates the user directly to the details page of the specific monument[cite: 361, 363].

### 🔊 Accessibility & Details
* [cite_start]The details page includes an "Listen" button that utilizes Text-to-Speech (TTS) to narrate the POI description[cite: 364, 367].
* [cite_start]Integrated web view to load external resources (e.g., Wikipedia links) directly within the app[cite: 368, 371].

### 📊 Database & User Statistics
* [cite_start]A local database stores user information (name, age) and tracks usage counters (how many times each POI has been opened or listened to)[cite: 372, 373, 374, 375].
* [cite_start]Dedicated "Statistics" screen to view user data and engagement metrics[cite: 382, 383, 385].

### 🧠 User Engagement
* [cite_start]After viewing or listening to a POI, users can take a multiple-choice quiz (at least 3 questions) to test their knowledge, with a final score display[cite: 386, 387, 388, 390].

## Technical Implementation
* [cite_start]**Development Tools:** Developed using either MIT App Inventor or Android Studio[cite: 345].
* [cite_start]**Code Organization:** Built with a focus on reusable procedures and clean code practices to minimize repetition[cite: 393, 395, 396].
